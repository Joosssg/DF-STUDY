## **4.1  변수란 무엇인가? 왜 필요한가?**

컴퓨터는 연산된 결과 값을 직접 사용하게 된다면 치명 적인 오류가 발생하기 때문에 그 ```결과값이 저장된 곳을 가르키는 이름 즉, 이름표(견출지)나 집주소라고 생각하면 되고, 이때 이 이름표나 주소를 *변수라고 한다.```<br>
>📣 값이 아닌 값이 저장된 곳을 가르킨다는 걸 기억하자.

<br>

## **4.2  식별자**
변수, 함수, 클래스 등을 모두 값을 구별할수 있어 **식별자**라고 한다.

<br>

## **4.3 변수선언**

변수를 선언을 하면 변수의 존재를 알리고, 초기화(undefined)를 한다.

```js
//ES5
var Gil;

//ES6
let Hello;
const World; 

//var Gil = undifined;
//let Hello = undifined;
//const World = undifined;
```
<br>

## **4.4 변수 선언의 실행 시점과 변수 호이스팅**

변수 선언은 런타임 이전(소스코드 평가)에 실행된다. 이처럼 자바스크립트의 특징을 변수 호이스팅이라고 한다.
>📣 var, let, const, function, function*, class 모든 식별자가 호이스팅 된다.

<br>

## **4.5 값의 할당**

```js
var gameBTI = '아싸! 프로젝트 취소됨ㅋㅋㅋㅋㅋ';

//축약문 이지만 변수선언과 값의할당은 다른시점에서 실행된다.
```
<br>

## **4.6 값의 재할당**
```js
var gameBTI = '아싸! 프로젝트 취소됨ㅋㅋㅋㅋㅋ';
gameBTI = '잘 못 들었습니다? 다시 한다는 말씀이십니까?'
```
변수의값은 재할당 할 수 있으며 재할당 시 같은 메모리공간에 기존의 값을 지우는 것이 아닌 **새로운 곳에 값이 저장되며 변수는 새로운 공간을 가르킨다.**

<br>

## **4.7 식별자 네이밍 규칙**

* 문자, 숫자, 언더스코어(_), 달러($)를 포함 시킬 수 있다. (특수문자제외)
* 이름의 첫글자로 숫자가 될 수 없다.
* 예약어는 식별자로 사용할 수 없다.
