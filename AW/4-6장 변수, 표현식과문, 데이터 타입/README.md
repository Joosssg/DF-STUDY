# 4장 변수

## 4.1 변수란 무엇인가?
- 하나의 값을 저장하기 위한 공간의 이름

## 4.3 변수 선언

``` js
var exp // 중복 변수명 및 변수 선언 가능 쌉가능

//---------------es6---------------

let exp// 값만 재할당 가능

const exp// 전부 재할당 불가능 
```   

## 4.4 변수 선언의 실행 시점과 변수 호이스팅
- JS 엔진은 변수 선언을 포함한 **모든 선언문을 먼저** 실행!!
- 그래서 서순 바뀌어도 선언됨 (단, 할당은 먼저 안됨)

➡ &nbsp; 이게 **변수 호이스팅**(변수 끌어올리기)
> hoisting: 끌어 올리기; 들어올려 나르기.
> 
## 4.5 값의 할당 & 4.6 값의 재할당
> 변수의 값을 재할당하면 이전에 값이 저장되어 있던 메모리 공간을 지우고 덮어쓰는 것이 아닌 새로운 메모리 공간을 확보하고 그 공간에 값을 저장한다.
- 이렇게 비효율 적인 구조인 이유는 JS가 매니지드 언어라서 메모리 제어가 안되기 때문
- 쓰기만 가능한 게스트 계정 바이브
## 4.7 식별자 네이밍 규칙
1. 문자, 숫자, _, $ 사용 가능
2. 숫자로 시작하면 안됨!!!!!
3. 예약어 사용 금지!!!!!
4. 유니코드 문자도 쓰지마셈
5. 대소문자 구분함
- 변수나 일반 함수는 카멜케이스 => `firstName`
- 생성자 함수 클래스는 파스칼 => `FirstName`
> 생성자 함수란?
> 
> - 재사용 할수 있는 객체 생성 코드를 구현할때 쓰는 함수 (다양한 객체를 만들때)
> - 앞에 `new` 붙는거임

# 5장 표현식과 문
## 5.1 값
> 값은 더 이상 평가될 수 없는 하나의 식

## 5.2 리터럴
- 값을 표현할때 약속된 문자나 기호를 써서 표현하는 표기법
## 5.4 문
- 문: 최소 실행 단위
- 토큰: 문법적으로 더이상 나눌수 없는 코드의 기본 요소
> 문 = 토큰 + 토큰 + 토큰 ...
## 5.5 세미콜론과 세미콜론 자동 삽입 기능
- 중괄호 코드블록`{...}`은 ; 붙이지 않음
## 5.6 표현식인 문과 표현식이 아닌문
- 을 구별하는 방법은 변수에 넣어보기
```js
var x; //는 표현식인 문일까?

var foo = var x; // 아니네?


x = 100; //는 표현식인 문일까?

var foo = x = 100; // 오 맞네
```
> 완료값
> - 개발자 도구에서 표현식이 아닌 문을 실행하면 `undefined` 나옴

# 6장 데이터 타입
## 6.1 숫자타입
1. JS는 하나의 숫자 타입만 존재함: 실수
2. Infinity 출력 가능
3. 산술 불가`NaN` 출력 가능
## 6.2 문자열 타입
1. **작따`'`**, 큰따`"`, 백틱 ( ` ) 으로 감싸는데 걍 작따 쓰셈
2. 따옴표 안쓰면 키워드나 토큰으로 구별함
## 6.3 탬플릿 리터럴
- ES6 신기술
1. 줄바꿈 안됨: 하고싶으면 막 `\n` 이런거 넣어야함
2. 표현식 삽입 가능:  `my name is ${first} ${last}.`
## 6.5 undefined 타입
- 변수에 값이 없다는 것을 명시하고 싶을 경우 `null`할당
## 6.6 null 타입
- 무조건 소문자임 `null`
- 함수가 유효한 값을 반환하지 못할때 에러 말고 `null`을 반환함
## 6.9 데이터 타입의 필요성
1. 저장할 때 메모리 공간 크기 미리 정할라고
2. 불러올 때 메모리 크기 참조할라고
3. 2진수를 어떻게 해석할 지 볼라고 (i.e. 0100 0001 = A or 65)
## 6.10 동적 타이핑
- JS는 타입을 동적으로 할당함
